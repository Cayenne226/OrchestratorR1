# T4.6 简单/复杂题效率分组统计

## 基本信息
- **优先级**: P1
- **预计耗时**: 0.5 天
- **依赖**: W2 eval 数据
- **产出**: `figures/efficiency_grouping.pdf`, 统计表格
- **状态**: [ ] 未开始

## 目标
按题目难度分组统计平均轮数和成本，**量化证明模型学会了"简单题走捷径，复杂题精雕细琢"**。

## 分组方案

### 按数据源分组（自然分组）
- **Simple**: NQ, TriviaQA, PopQA
- **Multi-hop**: HotpotQA, 2WikiMultihop, MuSiQue
- **Code**: HumanEval, MBPP

### 按实际轮数分组（结果驱动）
- **1-turn**: 直接 1 步回答
- **2-turn**: 2 步（如 refiner + executor）
- **3+ turn**: 需要分解/验证

## 统计指标

| 分组 | Ours 轮数 | Ours 成本 | Ours F1 | Fixed 轮数 | Fixed 成本 | Fixed F1 |
|------|-----------|-----------|---------|------------|------------|----------|
| Simple | ~1.2 | ~$0.001 | ~55% | 6.0 | ~$0.008 | ~50% |
| Multi-hop | ~3.5 | ~$0.005 | ~50% | 6.0 | ~$0.008 | ~52% |
| Code | ~3.0 | ~$0.006 | ~45% | 6.0 | ~$0.010 | ~42% |

**关键对比**: Simple 题上我们用 1/8 的成本达到更好的 F1。

## 实现

```python
# analysis/efficiency_grouping.py
def compute_efficiency_stats(eval_json):
    groups = {"simple": [], "multi_hop": [], "code": []}
    source_to_group = {
        "nq": "simple", "triviaqa": "simple", "popqa": "simple",
        "hotpotqa": "multi_hop", "2wiki": "multi_hop", "musique": "multi_hop",
        "humaneval": "code", "mbpp": "code"
    }
    for r in eval_json["results"]:
        g = source_to_group[r["source"]]
        groups[g].append({"turns": r["n_turns"], "cost": r["total_cost"], "f1": r["f1"]})

    for g, items in groups.items():
        print(f"{g}: avg_turns={mean(turns):.1f}, avg_cost=${mean(costs):.4f}, f1={mean(f1s):.3f}")
```

## ⚠️ 双配置注释
- 效率分组统计**两个 pool 都做**:
  - Cheap pool: 主要展示效率优势（简单题 1 步 vs Fixed-Pipeline 6 步）
  - Matched pool: 对比 cheap pool 的效率模式（frontier worker 下是否仍走捷径？→ 如果是 → 策略 transfer）
- 如果 matched pool 下简单题仍 1 步完成 → 说明"走捷径"是 orchestrator 学到的策略，而非因 worker 太弱不得不简化
