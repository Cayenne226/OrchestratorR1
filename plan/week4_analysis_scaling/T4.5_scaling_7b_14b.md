## 14B 训练方案（可选）

### A100 80GB × 4 配置
```bash
# 14B 全参: 单卡显存不够全参，需要 FSDP FULL_SHARD
# 或者用 LoRA 大幅降低显存

accelerate launch --config_file training/accelerate_fsdp_a100_4gpu.yaml \
    training/train.py \
    --model_path models/Qwen2.5-14B-Instruct \
    --data_path data/train_mixed.jsonl \
    --output_dir checkpoints/orch_grpo_14b_seed1 \
    --use_lora True \
    --lora_r 64 --lora_alpha 128 \
    --num_generations 8 \
    --per_device_train_batch_size 2 \
    --seed 42
```

显存估算 (14B, LoRA, A100 80GB):
- 模型权重 (FSDP /4): ~7GB
- LoRA 参数: ~0.5GB
- 优化器 (仅 LoRA): ~1GB
- GRPO rollout (G=8): ~40GB
- **总计: ~50GB** ✓

## 关键发现方向

1. **编排能力与模型规模正相关**: 更大模型 → 更好的 agent 选择
2. **小模型也能编排**: 3B 已经能涌现基本自适应（证明方法的广泛适用性）
3. **边际收益递减?**: 7B→14B 的提升可能小于 3B→7B（编排瓶颈在 agent 质量而非 orchestrator）
