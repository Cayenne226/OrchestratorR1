## 基本信息

- **预计耗时**: 0.5 天

## 目标
展示训练过程中行为如何变化，证明 **自适应编排是 RL 驱动涌现的**，而非 SFT 预设。

## 数据来源
从 wandb 导出训练过程中的逐 step 指标，或者在训练时每 50 steps 在一组固定测试题上评估。

## 图表设计

### 图 A: 平均轮数随训练的变化（按难度分组）
```
Avg Turns ↑
   5 |                          ──── 多跳推理
     |              ────────────
   3 |  ───────────
     |                               ──── 代码
   2 |
     |  ────────────────────────────
   1 |                               ──── 简单 QA
     +────────────────────────────→ Training Steps
     0    500   1000   1500   2000
```
**关键**: 简单题的轮数从 SFT 后的 ~2 降到 ~1（学会走捷径），多跳题保持或增加到 3-4。

### 图 B: Agent 调用比例随训练的变化
```
Call %  ↑
  60% |  exec_cheap ────────────
      |             ──────── decomposer
  30% |   ─────────            ──── exec_strong
      |             ──────── critic
  10% | ─────────              ──── synthesizer
      |                        ──── refiner
      +────────────────────────→ Steps
```
**关键**: 早期所有 agent 调用比较均匀（探索），后期分化出主导 agent。

### 图 C: 奖励分布随训练的变化
```
Reward ↑
  1.0 |           ──── 简单题 reward
      |      ─────
  0.5 |  ────
      |               ──── 多跳题 reward
  0.0 | ──────────
      |
 -0.5 | ──
      +────────────────────────→ Steps
```

## 实现方案

### 方案 A: 从 wandb 导出（推荐）
```python
import wandb
api = wandb.Api()
run = api.run("your_project/run_id")
history = run.history()
# 提取 reward_mean, turns_mean, agent_distribution 等
```

### 方案 B: 训练时定期评估固定测试集
在 train.py 中添加:
```python
FIXED_TEST_SET = [
    {"input": "What is the capital of France?", "type": "simple"},
    {"input": "Who directed the film...", "type": "multi_hop"},
    {"input": "Write a function...", "type": "code"},
    # ... 共 30 条 (10 simple + 10 multi_hop + 10 code)
]

if step % 50 == 0:
    results = evaluate_fixed_set(model, FIXED_TEST_SET)
    wandb.log({"fixed_test/simple_turns": ..., "fixed_test/multihop_turns": ...})
```

**注意**: 如果方案 B 需要重新训练，则需在 W2 就加入这个 hook。优先检查 wandb 是否已有足够数据。

## 论文叙述

> Figure X shows the evolution of orchestration behavior during GRPO training.
> (a) Average turn count diverges by task complexity: simple queries converge to 1-step execution
> by step 500, while multi-hop tasks stabilize at 3-4 steps.
> (b) Agent call distribution shifts from uniform exploration to task-specialized patterns.
> (c) Both simple and complex task rewards improve, but at different rates.
> **This behavioral divergence emerges without any explicit curriculum or task-type labels in the reward.**
