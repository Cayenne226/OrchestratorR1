## Introduction 结构 (~1.5 页)

### Para 1: 背景 + 痛点
LLM 已经很强大，但单一 LLM 有局限 → 多 Agent 系统。
但现有多 Agent 框架依赖人工设计（AutoGen, CrewAI）→ 能否端到端学习编排策略？

### Para 2: 现有工作的进展与局限
- Router-R1: 证明"路由"可以 RL 学习，但仅选模型、无功能分工
- Conductor: 证明"编排"可以 RL 学习，但一次性生成完整 workflow（开环），无法中间调整
- AgentConductor: DAG 拓扑优化，但限于代码任务

### Para 3: 本文方法
OrchestratorR1: 反应式多 Agent 编排
- Orchestrator 逐步观察 agent 返回、逐步决策
- 6 种功能角色（refiner, decomposer, executor, critic, synthesizer）
- GRPO 端到端训练，单一奖励函数
- 涌现自适应行为谱

### Para 4: 核心发现
- 自适应编排策略从 RL 涌现（热力图 + 训练动态证据）
- 反应式 > 开环（adversarial 实验 + case study）
- 一个策略跨 QA/推理/代码三种模态泛化
- **编排策略从 cheap worker 零样本迁移到 frontier worker**（双配置实验证据）
- 显式 Pareto 优化

### Para 5: Contributions (编号列表)
1. 首次将 reactive multi-agent orchestration 建模为可学习 MDP
2. 证明自适应编排策略从单一 RL 目标涌现
3. **编排策略零样本迁移: 用低成本 worker 训练的策略在 frontier worker 下同样有效**
4. 三个 track + Pareto + adversarial 实验的全面验证

## Related Work 结构 (~1 页)

- 局限: 同质模型选择，无功能分工

### RL-Learned Agent Orchestration
- **Conductor** (ICLR'26): GRPO 训练，开环 workflow 生成，推理 SOTA
- **AgentConductor**: SFT+GRPO，DAG 拓扑，代码任务

- AutoGen, CrewAI, LangGraph: 手工设计
- CAMEL, MetaGPT: 角色扮演
- 与本文区别: 端到端 RL 学习 vs 人工设计

### Agentic Reinforcement Learning
- GRPO (DeepSeek-R1), iStar, IGPO
- Tool-use RL: ToolFormer, Agent-R1
- 本文贡献: 多 agent 编排（非单 agent tool-use）
