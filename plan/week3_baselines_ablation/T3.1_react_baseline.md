## 目标
实现标准 ReAct (Reason + Act) agent baseline，使用 Qwen2.5-7B-Instruct 作为 backbone，支持 tool-use 调用外部 LLM API。

## 为什么需要这个
- ReAct 是 tool-use agent 的标准范式
- 对比 ReAct vs OrchestratorR1: 单模型 tool-use vs 多 agent 编排
- 如果 OrchestratorR1 > ReAct → 证明多 agent 功能分工的价值

## ReAct 实现方案

### Prompt 设计
```
You are a helpful assistant. You can use the following tools:
- search(query): Search for information using an LLM API
- answer(text): Provide the final answer

For each step, output your reasoning and action:
Thought: [your reasoning]
Action: search("your query") or answer("your answer")
Observation: [result from the tool]
... (repeat until answer)
```

### 与 OrchestratorR1 的关键区别
| 维度 | ReAct | OrchestratorR1 |
|------|-------|----------------|
| Agent 数量 | 1 (统一 search) | 6 种功能角色 |
| 功能分化 | 无 | refiner/decomposer/critic/... |
| Tool 选择 | search 或 answer | call type="X" |
| 训练 | 零样本 (prompt 驱动) | SFT + GRPO |
| 底层 API | 统一调 gpt-4o-mini | 不同 agent 调不同 API |

### 实现
在 `eval/baselines.py` 中添加 `react_baseline()`:
