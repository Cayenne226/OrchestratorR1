## 详细方案

### A. w/o critic
**修改**: agent_registry.py 中移除 `critic` agent，system_prompt.py 中删除 critic 描述。
**预期影响**: 多跳推理 F1 下降 3-8%（无法验证中间结果质量），简单 QA 影响小。

```bash
# 训练
bash training/train_a100.sh \
    --model_path checkpoints/sft_warmup_7b \
    --output_dir checkpoints/ablation_no_critic \
    --disable_agents critic \
    --seed 42

# 评估
python eval/eval_orchestrator.py \
    --model_path checkpoints/ablation_no_critic/final \
    --data_path data/test_qa.jsonl \
    --disable_agents critic \
    --output eval/results/ablation_no_critic.json
```

### B. w/o decomposer
**修改**: 移除 `decomposer` agent。
**预期影响**: 多跳推理 F1 大幅下降 (5-15%)，模型被迫单步处理复杂问题。简单 QA 无影响。

```bash
bash training/train_a100.sh \
    --disable_agents decomposer \
    --output_dir checkpoints/ablation_no_decomp
```

### C. w/o refiner
**修改**: 移除 `refiner` agent。
**预期影响**: 含隐含引用的多跳问题 F1 下降，简单问题无影响。预计整体影响较小。

```bash
bash training/train_a100.sh \
    --disable_agents refiner \
    --output_dir checkpoints/ablation_no_refiner
```

### D. α=0 (无成本惩罚)
**修改**: 奖励函数中 α=0，不惩罚 API 成本。
**预期影响**: F1 可能略高（模型自由调用贵模型），但平均成本大幅上升。

```bash
bash training/train_a100.sh \
    --alpha 0.0 --beta 0.1 --gamma 0.15 \
    --output_dir checkpoints/ablation_alpha0
```

### E. Fixed-Pipeline (仅评估，不训练)
**修改**: 评估时强制模型走 refiner → decomposer → executor_strong → critic → executor_strong → synthesizer
**实现**: 忽略模型输出，按固定顺序调用 agent

```bash
python eval/baselines.py \
    --method fixed_pipeline \
    --data_path data/test_qa.jsonl \
    --output eval/results/ablation_fixed_pipeline.json
```

## 批量执行策略

如果 4×A100 一次只能训一个:
```
Day 1: A (w/o critic) 训练 + 评估      (~10h)
Day 2: B (w/o decomposer) 训练 + 评估   (~10h)
Day 3: C (w/o refiner) 训练 + D (α=0) 训练  (~20h, 或并行 2×2 GPU)
Day 3: E (Fixed-Pipeline) 评估 (无需训练, CPU即可)
```

如果有更多算力: A/B/C/D 可以 4 个并行（各用 1 张 A100 + LoRA 模式）。
