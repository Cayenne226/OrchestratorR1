# T3.6 α 敏感性 / Pareto 前沿

## 基本信息
- **优先级**: P1
- **预计耗时**: 2 天 (6 个 α 值的训练 + 评估)
- **依赖**: W2 SFT checkpoint
- **产出**: `eval/results/alpha_{0.0,0.1,0.3,0.5,0.7,0.9}.json`
- **状态**: [ ] 未开始

## 目标
扫描 α (成本权重) 从 0 到 0.9，绘制 cost vs F1 的 Pareto 前沿图。
这是我们 "显式成本优化" 贡献的核心证据。

## 实验设计

### α 值选择
| α | 预期行为 |
|---|---------|
| 0.0 | 无成本约束，自由调用贵模型，F1 最高但成本最高 |
| 0.1 | 轻微成本感知 |
| 0.3 | 默认值，平衡点 |
| 0.5 | 中度成本约束 |
| 0.7 | 强成本约束，倾向 cheap agent |
| 0.9 | 极度成本约束，几乎只用 executor_cheap |

### 固定其他超参
- β = 0.1, γ = 0.15 (不变)
- 训练数据、模型、steps 全部相同
- Seed = 42 (每个 α 只跑 1 seed，节省时间)

## 执行

### 方案 A: 每个 α 重新训练 GRPO（最公平但最贵）
```bash
for ALPHA in 0.0 0.1 0.3 0.5 0.7 0.9; do
    bash training/train_a100.sh \
        --model_path checkpoints/sft_warmup_7b \
        --output_dir checkpoints/alpha_${ALPHA} \
        --alpha $ALPHA --beta 0.1 --gamma 0.15 \
        --seed 42

    python eval/eval_orchestrator.py \
        --model_path checkpoints/alpha_${ALPHA}/final \
        --data_path data/test_qa.jsonl \
        --output eval/results/alpha_${ALPHA}.json
done
```
**时间**: 6 × 10h = ~60h。需要提前开始或用多机并行。

### 方案 B: 部分 α 跳过训练，仅调评估时的惩罚（不推荐但快）
不推荐，因为不同 α 应导致不同的策略。

### 方案 C: 折中 — 只训 4 个关键点
训 α = {0.0, 0.3, 0.6, 0.9}，4 个点足够画出 Pareto 趋势。
**时间**: 4 × 10h = ~40h。

**推荐方案 C**，如果时间不够再减到 3 个点 (0.0, 0.3, 0.9)。

## Pareto 图预期

```
F1 ↑
 |  ● α=0.0 (高F1, 高成本)
 |   ● α=0.1
 |    ● α=0.3 ← Pareto 最优区域
 |     ● α=0.5
 |       ● α=0.7
 |         ● α=0.9 (低F1, 低成本)
 |
 +—————————————————→ Avg Cost ($)
   low                     high

对比:
 ★ Direct-GPT-4o: 一个固定点 (高F1, 高成本)
 ★ Router-R1: 一个固定点
 ★ ReAct: 一个固定点
```

**关键论点**: 我们提供一条 Pareto 前沿，而竞品只是固定点。用户可以根据预算选择 α。
