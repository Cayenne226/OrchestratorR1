# T1.5 A100 训练环境配置

## 基本信息
- **优先级**: P0
- **预计耗时**: 1-2 天
- **依赖**: 无
- **产出**: 可运行的训练环境 + 配置文件
- **状态**: [ ] 未开始

## 目标
在 A100/V100 80GB 机器上配置完整的训练环境，确保 7B 模型 FSDP 训练可以启动。

## 环境需求

### 硬件
- 4×A100 80GB 或 4×V100 80GB
- NVLink/PCIe 互联
- 至少 200GB 磁盘（模型 + 数据 + checkpoints）

### 软件栈
```
Python 3.10+
PyTorch 2.4.x (< 2.5.0)
transformers 4.44.x - 4.49.x
trl 0.13.x - 0.15.x
accelerate 0.34.x - 0.35.x
peft (LoRA, 备用)
flash-attn 2.x (pip install flash-attn --no-build-isolation)
datasets
deepspeed (备用)
openai
wandb
```

## 实现步骤

### Step 1: 基础环境
```bash
# 创建 conda 环境
conda create -n orch_r1 python=3.10 -y
conda activate orch_r1

# 安装 PyTorch (CUDA 12.1)
pip install torch==2.4.0 torchvision torchaudio --index-url https://download.pytorch.org/whl/cu121

# 安装项目依赖
cd OrchestratorR1
pip install -r requirements.txt

# 安装 flash-attn (A100 支持)
pip install flash-attn --no-build-isolation
```

### Step 2: 下载模型
```bash
# Qwen2.5-7B-Instruct (主实验)
huggingface-cli download Qwen/Qwen2.5-7B-Instruct --local-dir models/Qwen2.5-7B-Instruct

# Qwen2.5-3B-Instruct (消融对比)
huggingface-cli download Qwen/Qwen2.5-3B-Instruct --local-dir models/Qwen2.5-3B-Instruct

# (可选) Qwen2.5-14B-Instruct (scaling 实验)
# huggingface-cli download Qwen/Qwen2.5-14B-Instruct --local-dir models/Qwen2.5-14B-Instruct
```

### Step 3: 创建 A100 FSDP 配置
```yaml
# training/accelerate_fsdp_a100_4gpu.yaml
compute_environment: LOCAL_MACHINE
distributed_type: FSDP
fsdp_config:
  fsdp_auto_wrap_policy: TRANSFORMER_BASED_WRAP
  fsdp_sharding_strategy: FULL_SHARD        # ZeRO-3 (A100 显存充裕)
  fsdp_transformer_layer_cls_to_wrap: Qwen2DecoderLayer
  fsdp_backward_prefetch: BACKWARD_PRE
  fsdp_state_dict_type: SHARDED_STATE_DICT
mixed_precision: bf16
num_processes: 4
num_machines: 1
```

### Step 4: 更新训练超参数
```python
# training/train.py 中 A100 适配的超参数
A100_CONFIG = {
    "num_generations": 16,          # G=16 (vs 3090的G=8)
    "per_device_train_batch_size": 4,  # (vs 3090的2)
    "gradient_accumulation_steps": 4,  # (vs 3090的8)
    "max_prompt_length": 8192,      # (vs 3090的4096)
    "max_response_length": 2048,    # (vs 3090的1024)
    "max_obs_length": 1024,         # (vs 3090的600)
    "learning_rate": 1e-6,
    "bf16": True,
    "flash_attn": True,
}
```

### Step 5: 冒烟测试
```bash
# 1. 测试模型加载
python -c "from transformers import AutoModelForCausalLM; m = AutoModelForCausalLM.from_pretrained('models/Qwen2.5-7B-Instruct', torch_dtype='bfloat16'); print('OK')"

# 2. 测试 FSDP 启动（空跑 1 step）
accelerate launch --config_file training/accelerate_fsdp_a100_4gpu.yaml \
    training/train.py \
    --model_path models/Qwen2.5-7B-Instruct \
    --data_path data/train_small.jsonl \
    --output_dir checkpoints/smoke_test \
    --max_steps 1

# 3. 测试推理
python inference/infer.py \
    --model_path models/Qwen2.5-7B-Instruct \
    --api_base YOUR_API_BASE --api_key YOUR_API_KEY \
    --input "What is 2+2?"

# 4. 测试单元测试
python test_local.py --skip_api
```

### Step 6: 创建训练启动脚本
```bash
# training/train_a100.sh
#!/bin/bash
# A100 4×80GB FSDP 训练脚本
export CUDA_VISIBLE_DEVICES=0,1,2,3
export TOKENIZERS_PARALLELISM=false

MODEL_PATH=${MODEL_PATH:-"models/Qwen2.5-7B-Instruct"}
DATA_PATH=${DATA_PATH:-"data/train_mixed.jsonl"}
OUTPUT_DIR=${OUTPUT_DIR:-"checkpoints/orchestrator_r1"}
SEED=${SEED:-42}

accelerate launch \
    --config_file training/accelerate_fsdp_a100_4gpu.yaml \
    training/train.py \
    --model_path $MODEL_PATH \
    --data_path $DATA_PATH \
    --output_dir $OUTPUT_DIR \
    --seed $SEED \
    --api_base $API_BASE \
    --api_key $API_KEY \
    --alpha 0.3 --beta 0.1 --gamma 0.15 \
    "$@"
```

## 显存估算 (单卡 A100 80GB, 7B 模型)

| 组件 | 大小 |
|------|------|
| 模型权重 (FSDP FULL_SHARD, /4) | ~3.5GB |
| 优化器状态 (/4) | ~7GB |
| 梯度 (/4) | ~3.5GB |
| GRPO rollout (G=16, max_len=2048) | ~40GB |
| Activations + buffer | ~15GB |
| **总计** | **~69GB** ✓ |
