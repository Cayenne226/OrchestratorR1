## 基本信息

- **产出**: `data/train_code.jsonl`, `data/test_code.jsonl`

## 目标
加载 HumanEval 和 MBPP 代码基准，生成包含 test_cases 的统一 JSONL 格式。

## 数据集清单

| 数据集 | HF 路径 | 训练量 | 测试量 | 说明 |
|--------|---------|--------|--------|------|
| HumanEval | `openai/openai_humaneval` | 0 (仅测试) | 164 | 函数级代码生成 |
| MBPP | `google-research-datasets/mbpp` | 374 (train split) | 427 (test split) | 短程序 |

## 统一格式

```jsonl
{
  "input": "def has_close_elements(numbers: List[float], threshold: float) -> bool:\n    \"\"\"Check if...\"\"\"\n",
  "answer": "def has_close_elements(...): ...",
  "source": "humaneval",
  "task_id": "HumanEval/0",
  "test_cases": "assert has_close_elements([1.0, 2.0], 0.5) == False\nassert ...",
  "difficulty": "code"
}
```

## 实现步骤

1. **使用现有 `data_process/prepare_code.py`**:
   - 检查是否支持 train/test split
   - 确保 test_cases 字段完整（用于 Pass@1 评估）
   - 添加 `difficulty: "code"` 字段

2. **生成**:
   ```bash
   python data_process/prepare_code.py \
       --source humaneval,mbpp --split test \
       --output data/test_code.jsonl

   python data_process/prepare_code.py \
       --source mbpp --split train \
       --output data/train_code.jsonl
   ```

3. **验证**:
   - HumanEval 164 条测试
   - MBPP 427 条测试 + 374 条训练
   - 所有条目有 test_cases
