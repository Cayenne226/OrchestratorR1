# T2.3 GRPO 主训练 Seed 2, 3

## 基本信息
- **优先级**: P0
- **预计耗时**: 8-12h × 2 = 16-24h
- **依赖**: T2.1 (SFT checkpoint), T2.2 (确认训练可行)
- **产出**: `checkpoints/orch_grpo_7b_seed2/final/`, `checkpoints/orch_grpo_7b_seed3/final/`
- **状态**: [ ] 未开始

## 目标
重复 GRPO 训练两次（不同 random seed），用于报告 mean ± std。NeurIPS 要求统计显著性。

## 训练命令

```bash
# Seed 2
SEED=123 bash training/train_a100.sh \
    --model_path checkpoints/sft_warmup_7b \
    --output_dir checkpoints/orch_grpo_7b_seed2 \
    --seed 123

# Seed 3
SEED=456 bash training/train_a100.sh \
    --model_path checkpoints/sft_warmup_7b \
    --output_dir checkpoints/orch_grpo_7b_seed3 \
    --seed 456
```

## 执行策略

### 如果单机 4×A100:
- Seed 2 和 Seed 3 串行运行
- 总耗时: ~16-24h
- 在 Seed 2 训练时，可用 CPU 跑 Seed 1 的评估

### 如果有多机:
- Seed 2 和 Seed 3 并行
- 总耗时: ~8-12h

## 超参数
与 T2.2 完全相同，只改 seed 和 output_dir。

**重要**: 不要根据 Seed 1 的结果调参。3 个 seed 用相同超参数才能报 mean ± std。

## 验收标准
- [ ] `checkpoints/orch_grpo_7b_seed2/final/` 存在
- [ ] `checkpoints/orch_grpo_7b_seed3/final/` 存在
- [ ] 3 个 seed 的 wandb 曲线趋势一致（排除偶然性）
- [ ] 如果某个 seed 训崩了，重跑一次用新 seed
