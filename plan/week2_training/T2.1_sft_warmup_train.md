# T2.1 SFT 热身训练

## 基本信息
- **优先级**: P0
- **预计耗时**: 30-60 min (A100)
- **依赖**: W1 全部完成（数据 + 环境 + SFT 数据）
- **产出**: `checkpoints/sft_warmup_7b/`
- **状态**: [ ] 未开始

## 目标
用 200 条 SFT 数据微调 Qwen2.5-7B-Instruct，使其掌握 `<think>` → `<call type="X">` → `<answer>` 的输出格式。

## 训练配置

```bash
accelerate launch --config_file training/accelerate_fsdp_a100_4gpu.yaml \
    training/sft_warmup.py \
    --model_path models/Qwen2.5-7B-Instruct \
    --data_path data/sft_warmup.jsonl \
    --output_dir checkpoints/sft_warmup_7b \
    --num_train_epochs 3 \
    --learning_rate 2e-5 \
    --per_device_train_batch_size 4 \
    --gradient_accumulation_steps 2 \
    --max_seq_length 1024 \
    --bf16 True \
    --logging_steps 10 \
    --save_strategy epoch
```

## 超参数说明

| 参数 | 值 | 说明 |
|------|-----|------|
| epochs | 3 | 200 条数据，3 epoch 足够记住格式 |
| lr | 2e-5 | SFT 标准值，比 GRPO 的 1e-6 高一个量级 |
| batch_size | 4 × 4 GPU × 2 accum = 32 | 有效 batch 32 |
| max_seq_length | 1024 | SFT 示例不会很长 |

## 训练后验证

### Step 1: 快速推理测试
```bash
python inference/infer.py \
    --model_path checkpoints/sft_warmup_7b \
    --api_base YOUR_API_BASE --api_key YOUR_API_KEY \
    --input "What is the capital of France?"
```
**预期**: 输出包含 `<think>` 和 `<call type="executor_cheap">` 和 `<answer>`

### Step 2: 批量格式检查
```python
# 在 10 条不同类型的问题上测试
test_inputs = [
    "What is 2+2?",                                    # 简单事实
    "Who was president when the Berlin Wall fell?",     # 多跳
    "Write a function to reverse a string",             # 代码
    "Tell me about quantum entanglement",               # 开放问题
    # ... 更多不同类型
]
# 检查: format valid rate 应该 > 80%
```

### Step 3: Agent type 覆盖检查
- 测试问题应触发不同 agent type
- 简单题 → executor_cheap ✓
- 复杂题 → decomposer ✓
- 模糊题 → refiner ✓

## 常见问题与解决

| 问题 | 原因 | 解决 |
|------|------|------|
| 输出不包含 `<call>` 标签 | SFT 数据质量差或训练不足 | 增加 epoch 到 5；检查 SFT 数据格式 |
| 总是生成同一种 agent type | SFT 数据 agent 分布不均 | 重新平衡 SFT 数据中的 agent type 比例 |
| 输出截断 | max_seq_length 太短 | 增加到 2048 |
| loss 不下降 | lr 太低或数据格式问题 | 检查数据 tokenization；尝试 lr=5e-5 |
