# T2.2 GRPO 主训练 Seed 1

## 基本信息
- **优先级**: P0
- **预计耗时**: 8-12 小时 (4×A100)
- **依赖**: T2.1 (SFT checkpoint)
- **产出**: `checkpoints/orch_grpo_7b_seed1/final/`
- **预计 API 成本**: ~$3-5
- **状态**: [ ] 未开始

## 目标
从 SFT checkpoint 出发，用 GRPO 训练 Orchestrator 学会自适应编排策略。这是整个项目最核心的训练。

## 训练命令

```bash
SEED=42 API_BASE=YOUR_API_BASE API_KEY=YOUR_API_KEY \
bash training/train_a100.sh \
    --model_path checkpoints/sft_warmup_7b \
    --data_path data/train_mixed.jsonl \
    --output_dir checkpoints/orch_grpo_7b_seed1 \
    --seed 42
```

## 核心超参数

| 参数 | 值 | 说明 |
|------|-----|------|
| base model | checkpoints/sft_warmup_7b | SFT 预热后的 checkpoint |
| num_generations (G) | 16 | 每个 prompt 采样 16 条轨迹 |
| per_device_batch_size | 4 | 每 GPU 4 个 prompt |
| gradient_accumulation | 4 | 有效 batch = 4×4×4 = 64 prompts |
| learning_rate | 1e-6 | GRPO 标准低学习率 |
| warmup_ratio | 0.05 | 前 5% steps 线性 warmup |
| max_turns | 6 | 最多 6 轮 agent 调用 |
| max_prompt_length | 8192 | A100 可支持长上下文 |
| max_response_length | 2048 | 每轮模型生成上限 |
| max_obs_length | 1024 | Agent 返回截断长度 |
| α (cost) | 0.3 | 成本惩罚权重 |
| β (turns) | 0.1 | 轮数惩罚权重 |
| γ (efficiency) | 0.15 | 效率奖励权重 |
| total_steps | ~2000 | 约 5000 条 × 多 epoch |
| save_freq | 200 | 每 200 steps 保存 checkpoint |
| eval_freq | 200 | 每 200 steps 在验证集上评估 |

## 训练数据
`data/train_mixed.jsonl` = train_qa.jsonl + train_code.jsonl 混合
- QA 数据: ~6000 条 (6 datasets × 1000)
- 代码数据: ~374 条 (MBPP train)
- 总计: ~6374 条

## 监控指标 (wandb)

### 必须跟踪的曲线
1. **reward/mean**: 整体奖励均值 → 应持续上升
2. **reward/std**: 组内方差 → 应逐渐减小（策略趋向确定性）
3. **format_valid_rate**: 格式正确率 → 应保持 > 90%
4. **f1/mean**: 答案 F1 均值 → 应持续上升
5. **cost/mean**: 平均 API 成本 → 应趋向稳定或下降
6. **turns/mean**: 平均轮数 → 应区分简单/复杂题
7. **agent_distribution**: 各 agent type 被调用的频率 → 应随训练涌现差异

### 健康信号
- Step 100: format_valid_rate > 85%, reward > -0.5
- Step 500: f1 > 0.3, 简单题 turns < 2.0
- Step 1000: f1 > 0.5, agent 分布开始分化
- Step 2000: f1 > 0.6, 明显的 task-dependent 行为

### 不健康信号 ⚠️
- format_valid_rate 快速下降 (< 70%) → SFT 效果被冲走，需降低 lr
- reward 震荡不收敛 → G 不够大或 lr 太高
- 所有题都调同一个 agent → 奖励信号太弱，需增大 α 或 γ
- cost/mean 持续上升 → α 太小

## 训练中检查点

| Step | 检查项 | 如果异常 |
|------|--------|---------|
| 50 | GPU 显存稳定，无 OOM | 减小 G 或 batch_size |
| 200 | 第一个 checkpoint + 快速推理测试 | 检查输出格式是否合理 |
| 500 | wandb 曲线趋势判断 | 决定是否调参或重启 |
| 1000 | 中期评估（验证集 F1） | 与 base model 对比确认有提升 |
| 2000 | 最终 checkpoint | 完整评估 |
