The proliferation of large language models (LLMs) with diverse capabilities has motivated the development of intelligent routing mechanisms that dynamically assign user queries to appropriate models. Recent work, exemplified by Router-R1 (Zhang et al., 2025), demonstrates that reinforcement learning can train an LLM-based router to perform multi-round routing and aggregation—interleaving internal reasoning with external model invocations to tackle complex queries. While Router-R1 marks a significant advance over single-round, one-to-one routing paradigms, it operates under a fundamental architectural constraint: the routing target is always a monolithic LLM that receives the same query and independently produces a complete answer. This "route-to-model" paradigm, regardless of how many rounds it employs, treats each invocation as a black-box answer attempt, lacking the ability to decompose tasks, refine ambiguous inputs, verify intermediate results, or synthesize partial solutions—capabilities that are essential for complex, multi-step reasoning.

We argue that the natural evolution beyond routing is orchestration: rather than selecting which model answers a question, an orchestrator decides what functional role each invocation serves within a structured problem-solving workflow. Consider a multi-hop question such as "What is the GDP of the country where the inventor of the telephone was born?". A router can only dispatch this question wholesale to one or more LLMs and hope for a correct answer. An orchestrator, by contrast, can decompose the question into sub-tasks (identify the inventor → find the birth country → look up GDP), assign each sub-task to an appropriate executor, and synthesize the results—a qualitatively richer form of coordination.

In this paper, we present OrchestratorR1, a framework that trains a small language model (Qwen2.5-3B/7B) to serve as an adaptive orchestrator over a pool of six specialized agents: refiner (input clarification), decomposer (task decomposition), executor_cheap and executor_strong (task execution at different capability-cost tiers), critic (intermediate result verification), and synthesizer (multi-result aggregation). Crucially, the orchestration strategy is not hand-designed as a fixed pipeline or a rule-based control flow (as in AutoGen or CrewAI). Instead, we formulate agent orchestration as a Markov Decision Process and train the orchestrator end-to-end with Group Relative Policy Optimization (GRPO), using a unified reward that balances answer quality, API cost, and interaction efficiency.

A key finding of our work is that adaptive orchestration strategies emerge spontaneously from RL training under a single reward function. Without any explicit curriculum or task-type labels, the trained orchestrator learns to: (1) directly invoke a cheap executor for simple factual questions (1 round); (2) first refine ambiguous inputs before execution (2 rounds); (3) decompose multi-hop questions into sub-tasks, execute them in sequence, and synthesize results (3–5 rounds); and (4) employ strong executors with critic-in-the-loop verification for complex reasoning or code generation (4–6 rounds). This behavioral spectrum—from minimalist shortcuts to elaborate multi-agent workflows—constitutes emergent complexity that is qualitatively absent from routing-only approaches.

Our framework also substantially broadens the task scope of RL-trained LLM coordination. While Router-R1 is evaluated exclusively on seven QA benchmarks, OrchestratorR1 extends to code generation (HumanEval, MBPP) and demonstrates that the same orchestration policy generalizes across fundamentally different task modalities. Furthermore, by adopting GRPO instead of PPO, we eliminate the need for a separate critic network and vLLM-based rollout infrastructure, reducing training complexity while maintaining competitive performance on 4×RTX 3090 GPUs.

大型语言模型（LLM）的迅速涌现催生了智能路由机制的发展——动态地将用户请求分配给最合适的模型。近期工作，以 Router-R1（Zhang et al., 2025）为代表，证明了强化学习可以训练一个基于 LLM 的路由器执行多轮路由与聚合：在内部推理与外部模型调用之间交替进行，以应对复杂查询。尽管 Router-R1 相比单轮一对一路由范式取得了显著进展，其仍受到一个根本性架构约束——路由目标始终是一个接收相同问题并独立生成完整答案的单体 LLM。这种"路由到模型"的范式，无论经历多少轮次，本质上都将每次调用视为一次独立的黑盒答题尝试，缺乏分解任务、精化模糊输入、验证中间结果或综合局部解答的能力——而这些能力对于复杂、多步骤推理任务至关重要。

我们认为，路由的自然演进方向是编排（Orchestration）：与其选择由哪个模型来回答问题，编排器更关注每次调用在结构化问题求解流程中承担什么功能角色。以多跳问题"电话发明者出生地的国家 GDP 是多少？"为例：路由器只能将整个问题发给某个或某几个 LLM，寄希望于获得正确答案；而编排器则可以将问题分解为子任务（确认发明者 → 查找出生国 → 检索 GDP），将每个子任务分配给适合的执行器，最终综合各步结果——这是一种在质量层面上远超路由的协作形式。

本文提出 OrchestratorR1，一个将小型语言模型（Qwen2.5-3B/7B）训练为自适应编排器的框架，其调度对象是一组六类专业 Agent：refiner（输入澄清）、decomposer（任务分解）、executor_cheap 与 executor_strong（不同能力与成本层级的任务执行）、critic（中间结果验证）以及 synthesizer（多结果聚合）。关键在于，编排策略不是人工设计的固定流水线或基于规则的控制流（如 AutoGen 或 CrewAI），而是将 Agent 编排建模为马尔可夫决策过程（MDP），并使用组相对策略优化（GRPO）进行端到端训练，以平衡答案质量、API 成本与交互效率的统一奖励函数作为学习信号。

本工作的核心发现是：自适应编排策略在单一奖励函数驱动的 RL 训练过程中自发涌现。无需任何显式课程设计或任务类型标签，训练后的编排器能够自主学会：（1）对简单事实问题直接调用廉价执行器（1 轮）；（2）对模糊输入先精化再执行（2 轮）；（3）将多跳问题分解为子任务，依序执行并综合结果（35 轮）；（4）对复杂推理或代码生成任务，启用强执行器并引入 critic 循环验证（46 轮）。这一从极简捷径到精细多 Agent 工作流的行为谱系，是仅凭路由方法根本无法实现的涌现复杂性。

本框架还大幅拓展了 RL 训练型 LLM 协作的任务覆盖范围。Router-R1 仅在七个 QA 基准上进行评估，而 OrchestratorR1 进一步支持代码生成任务（HumanEval、MBPP），并证明同一编排策略可在本质上不同的任务模态间泛化。此外，采用 GRPO 替代 PPO，消除了对独立 Critic 网络和 vLLM 推理引擎的需求，在保持竞争性性能的同时显著降低了训练复杂度，全流程可在 4 张 RTX 3090 上完成。
